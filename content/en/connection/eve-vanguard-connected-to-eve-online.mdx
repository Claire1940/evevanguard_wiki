<Section icon={Sparkles} title="The Quasar Backend: How the Connection Works">

The link between EVE Vanguard and EVE Online is made possible by CCP Games' custom backend infrastructure, known as **Quasar**. Although both games run on completely different engines—EVE Online on its legacy codebase and EVE Vanguard on Unreal Engine 5—they communicate in near-real-time using an Event-Driven Architecture (EDA).

When a Warclone completes an objective on a hostile planet, the Vanguard backend acts as a trusted publisher. It sends structured data packets directly to Quasar via a highly secure, private network using gRPC and protocol buffers. 

| Backend Component | Primary Function | Impact on Players |
| :--- | :--- | :--- |
| **Vanguard Client** | Sends real-time combat and contract event data | Tracks player actions on the ground |
| **Quasar Gateway** | Processes incoming events and securely verifies game states | Bridges the two independent game engines |
| **EVE Online Services** | Aggregates data into active Faction Warfare systems | Shifts system-wide corruption or suppression levels |

<CardGrid cols={2}>
  <Card title="Event-Driven Architecture">
    - **Scalable Design**: Allows both games to scale independently without affecting server stability.
    - **Decoupled Systems**: State changes (like character undocking or contract completion) are published onto a central message bus.
    - **Real-Time Updates**: Ground actions update active Faction Warfare stats in space within milliseconds.
  </Card>
  <Card title="Secure gRPC Integration">

    - **Fast Communication**: Minimizes latency between planetary extractions and orbital market changes.
    - **Future-Proof**: Lays the technical foundation for two-way, real-time interactive events.
  </Card>
</CardGrid>

Once this gauge is filled, unique anomalies and high-value convoys appear in EVE Online space, creating immediate PvE and PvP opportunities for space pilots.
